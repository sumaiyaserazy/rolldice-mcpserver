### 3.2 Security Validation Pipeline

The demo visualizes a 6-step security validation process:

#### Step 1: Input Sanitization
- **Purpose:** Remove malicious characters and scripts
- **Validation:** Check for XSS, SQL injection patterns
- **Result:** Only clean, numeric input allowed
- **Status Indicator:** Real-time pass/fail feedback

#### Step 2: Schema Validation (Zod)
- **Purpose:** Enforce data type and range requirements
- **Validation:** Dice sides between 2-1000
- **Technology:** Zod schema validation
- **Result:** Type-safe, validated input

#### Step 3: Rate Limiting Check
- **Purpose:** Prevent abuse and DDoS attacks
- **Validation:** Check requests per minute against limit
- **Current Status:** 15/100 requests per minute
- **Action:** Block if exceeded, log attempt

#### Step 4: OAuth 2.0 Verification
- **Purpose:** Ensure authenticated, authorized access
- **Validation:** Token signature, expiration, scope
- **Token Details:** Scope: mcp:tools:execute | Expires: 2h 45m
- **Result:** Authentication confirmed

#### Step 5: MCP Tool Execution
- **Purpose:** Execute the dice rolling tool
- **Process:** Generate random number within validated range
- **Result:** Return result (e.g., rolled 4 on d6)
- **Performance:** <145ms average execution time

#### Step 6: Audit Logging
- **Purpose:** Track all tool calls for compliance
- **Data Logged:** Timestamp, user, input, result, IP
- **Storage:** Secure database with retention policy
- **Compliance:** SOC 2, GDPR ready
